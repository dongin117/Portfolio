아이튜버 크리에이터 PC버전 APP기획

I-TUBER Creator PC Application  

Step.4

Background. B2C타겟을 크리에이터로 1차 설정한 후 BJ나 YouTuber들이 손쉽게 이용할 수 있는 방안을 연구하였고, STEAM에서 서비스 중인 FaceRig를 벤치마킹하였다. 이 유틸은 내 얼굴을 캐릭터 얼굴로 보여주는 기능으로 제한되어 있었고, 공간이라는 부분에 초점을 맞추어 테마룸 형식에서 캐릭터를 이용하여 다양한 콘텐츠를 만들어 낼 수 있도록 구상하였다.

I-TUBER Creator PC Application

Overview. 나만의 특색있는 가상캐릭터로 테마룸에서 BJ나 유튜버를 넘어 일반인까지 손쉽게 콘텐츠를 만들 수 있도록 도와주며 얼굴노출을 하지 않고도 되는 버추얼유튜버로써 최적의 경험까지 만들어 주는 아이튜버 크리에이터 PC버전 어플리케이션이다. 

Date.       2020. 3 ~ 2020. 7 (단위별 진행중) 

Client.     두리번
Position.  PM, 서비스기획, UX디자인, 프로토타이핑 

Creator Platform Wireflow

1. 크리에이터 플랫폼 구상 

Description. 아이튜버 시스템을 완성한 다음 준비를 시작한 크리에이터 버전이다. 내부적으로 스튜디오에서의 한계에서 확장하여 기업이 아닌 일반유저들을 위한 버전을 준비했다. 우선 첫번째 Scrum으로 콘텐츠 플랫폼 형태로 설계하였다. 유저가 크리에이터버전으로 만들어진 콘텐츠를 공유하고 상대방이 만든 공간에서 같이 놀수 있게 하는 아이디어부터 시작하였다.

High Strategy Document

2. 크리에이터 상위문서 작성

Description. 기존 스튜디오 버전에서의 아쉬운 점과 개선점을 시작으로하여 대고객 서비스로의 확장에 필요한 기능을 세분화 하고 어느정도 전체 디자인을 포함한 디자인을 구체화 시키기 위해 상위 문서를 제작하였다. 지나고 나서 느끼는 거지만 확실히 초반 컨셉을 결정해 놓고 그걸 구체화 시켜놓으면 확실히 후반으로 가면서 정책이 흔들릴 일이 확실히 낮아진다는 것이다. 

Theme Room Concept UI draft 

3. 2D배경과 3D캐릭터 조합

Description. 전체 구성을 끝내고 난 다음 커스터마이징을 할 수 있는 캐릭터를 준비하였지만 만들어야 하는 유형과 기능이 개발 대비하여 너무 많은 시간과 비용이 소모되는 결과가 나와서 캐릭터에 의존하지 않는 다른 방법이 필요하였다. 그래서 우선순위를 캐릭터 커스터마이징에서 테마공간으로 변경하고 특정 3D 캐릭터와 2D공간을 조합하여 테마룸 컨셉을 만들게 되었다. 

Virtual Theme Room UI Design

4. 가상 테마룸 UI 디자인

Description. 테마룸으로 결정한 후 크리에이터의 콘텐츠를 분석해본 결과 음악, 취미, 교육, 투자, 재미, 등의 인기주제중에 글로벌 컨텐츠가 가능하면서도 누구나 할 수 있는 커버송 테마룸을 먼저 기획하였다. 기존 UI에서 창모드로 형태였는데 인터렉션을 포함한 감각적인 트랜지션을 사용하여 사용자가 직접 컨트롤함의 연속성 부여하고 다소 많은 메뉴와 툴을 쉽게 적응하도록 설계하였다. 또한 XD로 프로토타입을 만들어 직접 경험하고 주변 사람들의 피드백을 수집하고 반영하는 과정을 반복하였다.

Cover Song Theme Room Version

5. Cover Song Theme Room

Description.  음원을 가진 미디어와 협업을 준비하고 제휴된 음원을 커버하면 특정 재생UI와 노래가사가 나오게 기능추가를 하였다. 실제 유저는 가상캐릭터의 모습을 하고 본인의 목소리로 노래를 부르는 것이었고, 테마룸안에 있는 마이크와 스피커, 등의 실제 제품을 제휴하여 노출하는 전략으로 비지니스모델을 구성하였다. 해당 프로젝트는 몇단계 스크럼이 끝나고 개발 중에 있어 아직 영상으로 공개할 수 없는 점은 아쉽지만 완성이 되고 오픈이 되면 정말 기대되는 점이다.
